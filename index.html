20行でわかる！OpenMPWとOpenLANEとSkywaterPDKの様々

OpenMPWの概要

  • 無料でLSIを焼いてくれる
  • Caravel repo(RISC-V+IOの何か)のユーザースペースに実装する

必要要件

  • ライセンス諸々
  • Apache 2.0ライセンスのLICENEファイル
  • GDSIIレイアウト(再現可能)
  • Caravel repoのテストハーネスとパッドフレームの使用
  • OpenLANEフローのOpenMPWプリチェックツールでの合格

まとめ

  • Caravelをforkして改造
  • OpenLANEで通るならおｋ

各種リンク

  • OpenLANE : RTLからGDSII吐くやつ
  • Caravel : パッドフレーム
  • Caravel User Project
  • プリチェックツール
  • MPW-ONE

LSIを焼こう！(未完成)

概要はefablessの動画[1]を見れば大体わかる。また使うツールのバージョンやCaravelの概要はこのドキュメント[2]に書いてある。シャトル毎に形式が変わるのでMPW3の概要が出たら更新したい。

必要なもの

  • OpenLANE : RTLをGDSIIに変換する
  • Caravel_user_project : これのUser Project Areaに自分のデザインを加える

1.OpenLANEのインストール

Dockerは事前にインストールしてデーモンを起動しておく。cloneしてmakeするだけ。[3]






git clone https://github.com/The-OpenROAD-Project/OpenLane.git
cd openlane/
make openlane
make pdk
make test

2.Caravelのインストール

基本的に、forkしたCaravel_user_projectのリポジトリ[4]上で全てを進めていく

git clone git@github.com:efabless/caravel_user_project.git

3.CaravelのUser Project Areaに自分のデザインを追加する[5][6]

基本的に以下の流れで作る

  1. user_project_wrapperに自分のデザインを書き込む
  2. caravel_user_project/openlane/user_project_wrapper/config.tclを編集する

1.user_project_wrapperに自分のデザインを書き込む

caravel_user_project/verilog/rtl/user_project_wrapper.vのIOをそのままに中身を自作のデザインに置き換えると、ピン割り当ての設定ファイルをそのまま利用できるので非常に楽。[7]

CaravelのUser Project Areaは

  • SoC側とWishboneバスで繋がっている
  • SoC側と128本のデバッグ用の線で繋がっている
  • 外部と38本のIOポートで繋がっている

SoCとのデータ転送には基本的にWishboneバスを利用するので、Wishboneの仕様を各自で調べ、仕様に沿ったIOを持ったデザインを自分で作ってほしい。

GPIOに関してはUser Project Areaからは謎のテクノロジーでinput/output/inenb/outenbに分離されている[8]ように見えるので深く考えずに自分のデザインと繋げてよい。

Wishboneと接続するに当たり、ユーザーは自作のデザインにアドレスを0x30000000から0x80000000-1までで自由に割り当てできる。[9]

2. caravel_user_project/openlane/user_project_wrapper/config.tclを編集する

user_projecto_wrapper.vのみを編集する場合は特に何もしなくてよいが、自分のデザインは複数のVerilogファイルから構成される場合が殆どなので、GDSIIに変換する前にOpenLANEに追加したVerilogファイルを教えてやる必要がある。そのためにconfig.tclを編集する。[10] OpenMPWでは./verilog/rtl以下に自分のデザイン用のディレクトリを作るのを推奨している。[10:1]

config.tclを編集する訳だが、選択肢が２つある

  1. Insert Design macroに対応させる
  2. Flattening Designに対応させる

この２つの違いについてはここ[11]に解説を譲る。2の方法の方が最適化が効く(多分)のでこの方法を解説する。

以下はデフォルトのconfig.tclだが、




















set ::env(CLOCK_PORT) "user_clock2"
set ::env(CLOCK_NET) "mprj.clk"

set ::env(CLOCK_PERIOD) "10"

## Internal Macros
### Macro Placement
set ::env(MACRO_PLACEMENT_CFG) $script_dir/macro.cfg

### Black-box verilog and views
set ::env(VERILOG_FILES_BLACKBOX) "\
	$script_dir/../../caravel/verilog/rtl/defines.v \
	$script_dir/../../verilog/rtl/user_proj_example.v"

set ::env(EXTRA_LEFS) "\
	$script_dir/../../lef/user_proj_example.lef"

set ::env(EXTRA_GDS_FILES) "\
	$script_dir/../../gds/user_proj_example.gds"

4.OpenLANEでforkしたCaravelプロジェクトをGDSIIにする[13]

初めに環境変数を２つ追加してmakeする。




export OPENLANE_ROOT=<openlane-installation-path>
export OPENLANE_TAG=<latest-openlane-tag>
make openlane

以下のコマンドをforkしたcaravelリポジトリの直下で実行する



make precheck // プリチェックツールのインストール
